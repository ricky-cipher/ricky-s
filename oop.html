The DOM tree

As we said earlier, theDOM is an object-oriented reperesention of the web page which can be modified with a scripting langauge such as JavaScript. The DOM consists of nodes which is simply an HTML element.
With this, the Dom is like a 'tree structure' that represents carries all HTML elements.

The document acts as the root to the nodes.

Let's take an example of a skeletal view of a web page.
ie.

<html>
<head>
</head>
<body>
</body>
</html>
The above code represnts a full web page. The web page is the root and has nodes.
HTML is a node which holds two other nodes, head and body.
In this case, HTML is known as the Parent node. The head and body are under html hence they are children of html.
So it is right to say Html is a parent of head and head is a child of html.

Lets apply this in a list.

<ol id="list">
<li> first item </li>
<li> second item </li>
<li> third item </li>
</ol>

In the example above, 'ol' is the parent node of the three list items. The list items can be termed as children of 'ol' element.

We can manipulate the tree using javascript inbiult methods.
Let's grab and store the parent node in a variable, 'par';

var par = document.getElementById('list');
We just stored the parent element/node together with its children, but we are still on the top level of the parent node. To access the first child node of 'ol';
par.children[0];
Every node has a property, 'children' which is a pointer to an array-like structure containing the nodes. Just like we index array elements, child elements are indexed the same way( starting '0'). Therefore the second child of 'ol', will be accessed as follows;
par.children[1];

Each node has a firstChild an lastchild property.
ie.

par.firstChild.innerHTML;
will rerurn the value of the first element
par.lastChild.innerHTML;
will rerurn the value of the last element

Modifying the tree

Each and every node has the following methods;

  1. createElement()

    2. This method creates a new element/node. since we are creating a new element on the document, we use the document as an object;
    ie.

    var newEl= document.createElement('li');
    we created a new list element ('li') and stored it in a variable 'newEl'.
  2. appendchild()

    3. adds a child/node to the tree of children and makes its the last child/ it adds at the end of the list.

    Let's append the new element we created earlier to our list;
    par.appendChild(newEl);
  3. removechild()

    4. this method removes a node from the tree.

    Let's assume we stored the second node in a variable, 'sec' and we want to remove it from the tree.
    ie.
    var sec = par.children[1];
    par.removeChild(sec);

Inheritence

lets take an example, a child acquires certain traits from the parents. So whatever is in the parent, is also in the child. But the child can optionally add what is in it.
A class can acquire acquire members from another class and can optionally add more members of its own. The class to which its members are acquired from another class is known as the sub class or child class or derived class, whereas the class to which its members are acquired by another class is known as the super class or parent class

Specialization and Generalization

Let's take an example of system. A system contains other smaller systems of its own called sub systems. When analyzing a system as a whole, this is refferred to as generalization whereas one may decide to go into the finer details of the system and decide to work with single portion of the system/sub-system, this is referred to as specialiaztion

Now back to the classes, while dealing with the super class, we are generalizing the program since we are working with the larger program as a whole. As we come down the inheritence tree, we are specializing, since we are dealing with single smaller parts of a larger class.
ie. we have different shapes and each of these shapes have more types of their own. for instance there are different types of triangles, (right-angled, isosceles and equilateral trianles).

lets create a super class;
class shapes{
shape() {
return'this is a shape';
}

now lets derive a class from the class 'shapes'. Like we saw before, there are different shapes, such as rectangles, traingle, circles ... So the class 'shapes' can have many sub-classes of different types of shapes.
ie.

class rectangles{
rect extends shapes() {
return'this is a rectangle';
}

class circles{
circ extends shapes() {
return 'this is a circle';
}

class triangles{
tri extends shapes() {
return 'this is a triangle';
}

we have three classes. but we can decide to go more deeper into one of them by creating a sub-class that derives its members from the class.
lets use our latter example, 'triangles'

class rightAngled{
right extends triangles() {
return 'this is a right-angled triangle';
}

Lets try and understand this step by step:
At first, we started with a class shapes that looks at all the shapes in general.(remember generalization). The 'class shapes' was a super class As we come down to look at these types of shapes, we are specializing. Then we come down to different types of a single shape. Lower classes are deriing members from the super class or rather the class above them and also optionally adding other members. In that case, we are using the keyword extends on the sub-class to show that the class is a derivative of the super class defined. e.g

class right extends triangles()
this means the sub-class, 'right' is a derivative of class shapes.

But one may ask how these classes are connected or how they work. lets try to pu this in action;

ie. we want to define a shape which is a right-angled triangle. so we will need the statement returned in the super class, in the sub-class, rightAngled.
class shapes{
shape() {
return'this is a shape';
}

class triangles{
tri extends shapes() {
let sh = super.shape();
'return 'this is a triangle';
}

class rightAngled{
right extends triangles() {
let tr = super.tri();
return sh + ', ' + tr + ', ' + 'this is a right-angled triangle';
}

In this case we included a super class in the sub class by assigning it to a constant variable. by placing it as a constant, it means that the members of the super class will be present and will nit be changed. the child class can only add members of its own.

Remember to access a member of a class, you must craete an object of its own type.
